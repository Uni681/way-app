# WAY — アプリ仕様書 v2
_ドリンクバー・人間図鑑・ストック上限UX 決定反映_

---

## コンセプト

> 属性を脱ぎ捨て、脊髄で会話する24時間

「うぇい！」の一言で波長が合うかどうかだけを確認したい。  
スーパーの惣菜コーナーで目が合うような、たまたま隣り合わせた感。  
深夜のファミレスの空気をアプリの中に再現する。

---

## 絶対ルール（三無）

| ルール | 内容 |
|--------|------|
| 本名禁止 | コードネームのみ |
| 顔写真禁止 | プロフィール画像は自撮り・顔写真NG |
| 電話禁止 | 音声・ビデオ通話なし |

- 原則タメ口。敬語によるテンポの乱れを排除。
- AIによる自動BAN：30回未満での連絡先提示・「会おう」発言は即アカウントBAN。
- **30回達成後はすべてのルールから解放される。** 顔を送っても、他のSNSに移ってもいい。WAYの中で会う約束をしてもいい。どうするかは2人が決める。

## 画面構成

### 01 メイン画面（ホーム）

**投げる側**
- 卓上ベル型ボタンを押すとテキスト入力が開く
- 入力して「投げる」→ 待機状態へ
- 相手がマッチした瞬間チャットが始まる
- 投げた側は相手を選べない（完全ランダム）
- 投げたチャットは1本のみ同時進行

**受ける側（その日初めて開いたとき）**
- 声が丸く浮かんで最大3つ届いている
- 丸は選ぶ前「存在してるかしてないかの瀬戸際」→ 淵がぼんやり、少し透明
- 選んだ瞬間だけ輪郭が確定。選ばれなかった丸は消える
- 声が1つしかない場合は残り2つが「もうすぐ誰か来るよ」状態でふわふわ待機し、届き次第順番に切り替わる
- 声が0の場合も同様に3つ全部が待機状態から始まる

**タイムライン（ホーム下部）**
- タイムラインに流れてくる声を見て「乗る」を選べる
- 表示順：投げてからの経過時間が短い順（タイミングが近い順）

**再会通知**
- ランダムマッチで2回目以降の相手と繋がったとき、双方に表示
  - 例：「あなたたち、今日で3回目ですよ」
- 回数は会話開始時にさりげなく出す（押しつけない）

### 04 ストック機能

**ストックとは何か**
- ストック = チャット履歴の保存 + 再会確率のUP
- 相思相愛（両者がストック）になった瞬間から有効
- 次に再会したとき「続きから」始まる
- 片方のみのストックは有効にならない（片想いストックは存在しない）

**上限：最大5枠**

**ストック満枠時のUX（v1から更新）**
- 5枠が埋まっているときにノリが合う人と話した場合、「印（ブックマーク）」をつけられる
- 印をつけた人と再会したとき「この前印つけた人だ」と通知される
- ストック枠が空いたとき（誰かが解除したとき）、印をつけた人がいれば通知される
- 印に回数制限なし。何人につけてもいい
- 印だけではチャット履歴は保存されない（あくまでブックマーク）

**解除**
- どちらかが解除したらチャット履歴ごと消える
- 解除前に確認モーダル：「ほんとに消すの？この会話は戻りません。」
- 相手には通知される（「〇〇がストックを解除しました」）

**再会確率の仕組み**
- ストックなし・印のみ：確率変わらず（フラット）
- 相思相愛ストック：再会を重ねるごとに確率が上昇
- 相思相愛ストック×回数が多いほど急角度で上昇
- 30回達成：確率の概念から外れ、自由チャットへ

### 05 再会ガチャ

- ストック済みの相手と再会できる確率が累積で上がっていく
- 再会ごとに「何回目の再会か」「会話の一言メモ」が記録される
- ストックタブから各ユーザーの再会履歴・確率バーを確認できる
- オンライン状態の表示：「オンライン」「1日前」「3日前」など

### 06 人間図鑑

**コンセプト**
- クエストや達成条件なし。純粋なコレクション
- 誰かと会話するたびに、相手の成分タグ4軸が図鑑に積み上がる
- 「完走」= 1チャットで20往復以上のやりとり（仮値。テスト後に調整）
- 完走するたびに、その人の成分が図鑑のカウントに加算される

**図鑑の見え方（例）**
- 看護師：12回
- エンジニア：34回
- 映画好き：8回
- ENFP：5回
- 農家・漁師：1回（レア）

**設計の意図**
- 話したことのない職種・趣味の組み合わせが図鑑に空欄で見える
- 「この人まだ会ったことない」という探索欲が生まれる
- 希少な属性（農家、伝統芸能など）と話せるとレア感がある
- 選択肢は多ければ多いほど図鑑が豊かになる → 今後も継続して拡充

### 07 30回達成 → 自由チャット開放

**演出**
- 達成画面にレシートが表示される
  - 最初の出会いから何日
  - 相思相愛になった日
  - 一番長かった会話の時間
  - 使ったドリンクの種類と回数
  - 再会回数 30/30
- コピー：「この会話が、ずっとここにある」
- ボタン：「チャットを開く」

**自由チャット状態**
- ヘッダーのタイマーが消える（「タイマーなし」表示）
- 24時間リセットなし
- チャット履歴はそのまま継続（新しい部屋ではなく、今までの会話が開放された）
- WAYのすべてのルールから解放
  - 顔写真を送ってもいい
  - 他のSNSのアカウントを交換してもいい
  - WAYの中で会う約束をしてもいい
  - どうするかは完全に2人が決める

## BANシステム設計

**守る対象**
- 30回未満での連絡先交換・外部SNSへの誘導
- 「会おう」発言（30回達成前）
- それ以外のモラル違反（暴言・ハラスメント）は別レイヤーで対応

**3層構造**

_第1層：パターンマッチ（即時BAN）_
- キーワード・正規表現で機械的に弾く
- 対象例：「LINE教えて」「インスタは？」「@〜（SNSアカウント形式）」「会いたい」「どこ住んでる」「電話番号」「連絡先」
- 検出したら即アカウントBAN
- ひらがな・カタカナ表記のバリエーションも網羅する

_第2層：AI判定（警告→2回目でBAN）_
- 第1層をすり抜ける巧妙な表現をAIが文脈込みで判定
- 例：「れんらくさきこうかんしない？」「外で話そ」「他のとこで話さない？」
- 使用モデル：Claude API
- 判定結果がグレーの場合 → 警告メッセージを表示して続行
- 2回目の検出 → BAN
- 30回達成済みペアの会話は第2層の判定対象から外す

_第3層：ユーザー通報（人間レビュー）_
- チャット画面に通報ボタンを設置
- 通報が来た会話ログを運営がレビュー
- 悪質と判断した場合は手動BAN

**誤BAN対策**
- 即BANは第1層のみ。第2層は必ず警告を挟む
- BANされた側には理由を簡潔に通知する（「連絡先の共有はWAYのルール違反です」等）
- 異議申し立てフォームを用意する（初期はメール対応でも可）

## マッチングアルゴリズム

**基本方針：完全ランダム**
- 属性・MBTIによる相性考慮なし
- 同日同一人物との再マッチも排除しない純粋ランダム
- 「また会ったじゃん笑」もWAYの体験のひとつ
- ストック済みの相手との再会確率UPだけが唯一の例外

**ストック済みとの再会確率**
- 相思相愛ストックの回数が増えるほど確率が上昇
- 純粋ランダムのプールの中で、ストック相手が引かれやすくなるイメージ
- アルゴリズムの詳細パラメータは実装時に調整

## 通知設計

**必須（デフォルトON・ユーザーが切れない）**

| タイミング | 通知文（案） |
|-----------|-------------|
| 誰かが声に乗った＝チャット開始 | 「誰かが乗ってきた」 |
| 相思相愛成立 | 「相思相愛になった。ストックされた。」 |
| 24時間タイマー残り1時間 | 「あと1時間で消える」 |

**オプション（デフォルトOFF・ユーザーが設定から変更可）**

| タイミング | 通知文（案） |
|-----------|-------------|
| ストック相手との再会チャンス | 「〇〇と再会できるかも」 |
| 印をつけた人と再会できるタイミング | 「前に印つけた人が来てる」 |
| 30回達成 | 「30回、積み上がった」 |
| 相手がストックを解除した | 「〇〇がストックを解除しました」 |

**通知設計の原則**
- 文言は短く、WAYの空気感を保つ。「チャットが開始されました」みたいなシステム語は使わない
- 深夜帯（2時〜7時）はデフォルトで通知をミュート。設定で変更可
- プッシュ通知はアプリ内バナーと併用
